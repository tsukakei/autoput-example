# Informal Comment

## CUT's good points and bad points

good

- Easy to understand what function is called.

bad

- Length of code is likely to get longer.

## PUT's good points and bad points

good

- Easy to add new tests.

bad

- Becomes burdensome to check parameters and expected result of tests when there are many tests.
